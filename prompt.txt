**Phase 7 scope — Flashcards & Spaced Repetition:**

1. **Schema** — add `FlashcardSchema` and `DeckSchema` to `src/schema/entities.js`:
   - `Flashcard`: `{ id, deckId, front, back, created, nextReview, interval, easeFactor, repetitions }`
   - `Deck`: `{ id, courseId | null, title, created, cardCount }`
   - Use SM-2 algorithm fields (`interval`, `easeFactor`, `repetitions`) — no external library

2. **Store** — `src/store/flashcards.js`:
   - `getDecks()`, `getDeck(id)`, `writeDeck(deck)`, `deleteDeck(id)`
   - `getCards(deckId)`, `getCard(id)`, `writeCard(card)`, `deleteCard(id)`
   - `getDueCards(deckId)` — returns cards where `nextReview <= Date.now()`
   - All backed by `store/storage.js` adapter (same pattern as `content.js`)

3. **SM-2 algorithm** — `src/utils/sm2.js`:
   - Pure function `reviewCard(card, grade)` → returns updated card fields
   - `grade` is 0–5 (0–1 = fail, 2 = hard, 3 = good, 4 = easy, 5 = perfect)
   - Standard SM-2: update `easeFactor`, `interval`, `repetitions`, compute new `nextReview`
   - No side-effects — store write happens in the flashcards page, not here

4. **Flashcards page** — `src/ui/pages/flashcardsPage.js` mounted in `#pageFlashcards`:
   - Left sidebar: deck list with due-card count badges, "+ New Deck" button
   - Main area (two states):
     - **Deck overview**: card count, due count, "Study Now" button, card table (front truncated, next review date, delete)
     - **Study session**: full-screen flip card (click or Space to flip), grade buttons (Again / Hard / Good / Easy), progress bar showing cards remaining in session, "End session" exits back to overview
   - Flip card: CSS 3-D transform, front shows `card.front`, back shows `card.back`
   - Empty state if no decks yet (prompt to generate from a course or create manually)

5. **Generate cards from a course** — `src/ui/blocks/generateCardsModal.js`:
   - Triggered from the deck overview when `deck.courseId` is set, or from a "Generate" button in the deck list
   - Calls the existing `getAllLessonsForCourse(courseId)` and renders a checklist of lessons
   - User picks lessons, clicks "Generate" — for now, emit `bus.emit('cards:generateRequest', { deckId, lessonIds })` and show a placeholder toast ("Generation coming in Phase 8")
   - Modal lives in its own block file, not inside the page

6. **XP integration** — emit `bus.emit('xp:earn', { amount: 10 })` after each successful card review (grade ≥ 2) so the existing `onLessonComplete` / XP counter in `app.js` can credit it
